# GymBro

Android-приложение для составления планов тренировок и отслеживания прогресса с системой **15 уровней силы**.

## Особенности

- **15 уровней силы** (Новичок → Средний → Опытный → Элита), автоматический расчёт по лучшим результатам на 5 повторений в жиме, приседе и тяге за последние 6 месяцев.
- Готовые планы тренировок для каждого уровня подготовки + возможность создавать свои с нуля.
- Экран тренировки с логированием подходов, таймером отдыха и уведомлением о новом PR.
- Автоматическое обновление личных рекордов (1RM и 5RM) при каждом подходе.
- Графики прогресса по каждому упражнению (MPAndroidChart).
- Анимация «повышения уровня» с разблокировкой новых планов.
- База из ~25 популярных упражнений с поиском.

## Стек

- **Kotlin 2.0** + **Jetpack Compose** + **Material 3**
- **MVVM** + Repository + Use Cases
- **Room** (через KSP)
- **Hilt** — DI
- **Navigation Compose**
- **Coroutines + Flow**
- **MPAndroidChart** — графики
- Тёмная тема по умолчанию

## Экраны

| Экран | Назначение |
|---|---|
| Splash | Проверка онбординга |
| Onboarding | Ввод стартовых 5RM или пропуск (старт с 1 уровня) |
| **Dashboard** | Геройская карточка уровня + CTA «Начать тренировку» + разделы |
| Plans | Список готовых планов с фильтром Новичок/Средний/Опытный + заблокированные по уровню |
| PlanEditor | Создание своего плана: дни (3–6), упражнения, подходы/повторы/отдых |
| Exercises | База упражнений с поиском (также работает как picker из PlanEditor) |
| WorkoutSession | Логирование подходов + таймер отдыха + PR-баннер |
| Progress | Уровень + статистика + график по упражнению + список PR |

## Архитектура

```
data/
  local/entity/    — 8 Room-сущностей
  local/dao/       — DAO для каждой сущности
  local/converter/ — TypeConverters для enum
  repository/      — ExerciseRepository, WorkoutRepository, ProgressRepository, LevelRepository
  seed/            — LevelThresholds (таблица 15 уровней), ExerciseSeed, WorkoutPlanSeed, SeedRunner

domain/
  model/           — StrengthLevel, LevelTier, BigThreeLift, OneRmFormula
  usecase/         — CalculateStrengthLevel, ObserveCurrentLevel, LogSet, StartWorkout, Onboarding

di/                — Hilt-модули
ui/                — экраны Compose + ViewModels + components
```

### Ключевые места

- **`CalculateStrengthLevelUseCase`** — сердце системы прогрессии. Алгоритм:
  1. Для каждого из трёх лифтов берём max вес ×5 за окно (6 мес.).
  2. Определяем макс. достигнутый уровень по каждому лифту из `LevelThresholds`.
  3. Общий уровень = **минимум** по трём (чтобы подтягивать слабое звено).
  4. Прогресс к следующему = совокупный по всем трём, взвешенный по недобору.
- **`LevelThresholds.kt`** — таблица 15 уровней с опорными точками (1, 5, 10, 15) и линейной интерполяцией между ними.
- **`ProgressRepository.logSet()`** — автоматически детектит новые 1RM и 5RM PR при каждом подходе.
- **`LevelCard.kt`** — геройская градиентная карточка с tier-цветом (серый → синий → золотой → фиолетовый), крупным номером уровня, прогресс-баром и breakdown по каждому лифту.

## Запуск

1. Android Studio **Jellyfish** или новее, JDK 17.
2. Открыть проект → Sync Gradle.
3. **Важно:** архив не содержит `gradlew`/`gradle-wrapper.jar` (бинарные файлы). После открытия в Android Studio запустите в терминале:
   ```
   gradle wrapper --gradle-version 8.9
   ```
   или просто откройте проект — Android Studio предложит сгенерировать wrapper автоматически.
4. Добавьте иконки `ic_launcher` через **Image Asset Studio** (File → New → Image Asset).
5. Минимальный SDK: 26 (Android 8.0). Target: 34.
